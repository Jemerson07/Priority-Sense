# Priority-Sense — QA module checklist

## Dashboard
- overview metrics render correctly
- realtime activity panel renders with recent events
- operational insights render in three groups
- alerts and trends render side by side
- overall spacing and visual hierarchy remain consistent

## Inbox
- quick summary cards render correctly
- filter bar remains aligned on desktop and mobile
- ticket list renders without overflow
- selected ticket detail is readable and balanced
- priority badges and tags remain visually consistent

## Rules
- score bands render clearly
- rule cards show category, status and weight
- active/inactive states are distinct
- spacing remains consistent across cards

## Settings
- connector cards render status and action clearly
- environment variables panel remains readable
- status pills keep strong contrast
- code/value previews do not overflow badly

## Auth entry
- login shell is balanced on desktop and mobile
- field labels, placeholders and CTA remain clear
- layout stacks correctly on smaller screens

## Shared UI
- empty state is readable and centered enough
- loading state remains consistent with design language
- section card works as a reusable wrapper
